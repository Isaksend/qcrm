The system uses a client-server architecture organized into distinct layers to separate presentation, business logic, and data storage.


\section{Structural Modeling: Component Diagram}
The monolithic unit is structured into three main layers:
\begin{itemize}
    \item Presentation Layer: Handles the user interface and session validation using a mobile-first Vue.js application.
    \item Business Logic Layer: Manages core services and the integration of analysis modules. In this layer, FastAPI acts as the engine for asynchronous request handling and task management.
    \item Persistence Layer: Manages data storage using SQLAlchemy to map Python objects to the relational database, ensuring strict data integrity.
\end{itemize}

This prevents the user interface from freezing while a background task fetches the deal context, communicates with the machine learning provider, and updates the record with the resulting recommendation.

\subsection{Diagram 2: Atomic Lead Conversion}

This process manages the transformation of a lead into associated account, contact, and opportunity records. The system utilizes atomic transactions to ensure that the lead is only marked as converted if all new records are successfully created \cite{keogh2025sqlalchemy}. If any part of the operation fails, the transaction is rolled back to prevent data corruption.

\section{Process Modeling: BPMN Diagrams}

\subsection{Lead Scoring and Visualization}

This process ranks leads based on their conversion probability. When a new lead is created, the system calculates an internal score based on historical patterns. If the score meets a specific threshold, the system triggers a request for advanced analysis to prioritize the lead within the funnel.

\subsection{Churn Analysis}

This is a proactive retention process triggered by a scheduled timer. The system scans customer activity to identify inactivity patterns that match historical churn data. If a risk is detected, an alert is generated for the manager to initiate a retention workflow.
